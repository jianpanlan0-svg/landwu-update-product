# 上传器说明

## 结构

- 本地服务：负责登录态保存、任务执行、队列调度、报告生成。
- 本地网页：负责可视化操作、进度展示、历史任务展示。
- 脚本猫桥接：负责把当前领物网页登录态同步到本地服务。
- 三步脚本：分别处理上传图库、批量设计、TEMU 汇出。

## 三步流程

### 1. 上传图库

- 输入图片文件夹和任务标签。
- 批量上传图片到领物图库，并统一设置标签。
- 单张图片失败会自动重试 1 次。
- 重试后仍失败的图片会复制到 `上传失败_标签_店铺_公版` 目录，原图不删除。
- 全部上传成功后，原图片文件夹会自动标记为 `（已上传图库）`。

### 2. 批量设计

- 用标签筛选第一步上传的主题图。
- 用公版名称搜索并精确匹配设计公版。
- 支持“最大化设计”选项，默认不勾选。
- 接口直接生成成品，不依赖网页逐张预览渲染。

### 3. TEMU 汇出

- 用标签筛选成品列表。
- 选择店铺和汇出模板。
- 默认 SKC 规则为店铺名提取安全前缀 + 日期，不包含 `TEMU` 和中文。
- 支持多店铺：第一家跑完整流程，后续店铺只跑第三步汇出。

## 队列

- 可以立即执行当前任务，也可以加入任务队列。
- 如果任务正在运行，新增队列会在当前任务结束后自动续跑。
- 队列项显示文件夹、标签、执行步骤、公版、店铺。

## 报告与失败处理

- 每一步都会在 `reports/` 下生成 JSON 报告。
- 第一阶段失败报告包含失败文件、失败原因、复制目标、重试记录。
- 前端历史任务会展示成功、失败、店铺、标签和执行时间。
- `reports/` 是本地运行产物，不提交到 Git。

## 本地配置

- `auth-state-v1.json`：网页登录态，本地生成，不提交。
- `obs-config.json`：OBS 密钥，本地生成，不提交。
- `obs-config.example.json`：OBS 配置示例，可以提交。

## 常用命令

```bash
npm install
npm start
npm run build:exe
npm run build:mac-package
```

GitHub Actions 自动打包：

- 推送到 `main`：自动构建 Windows EXE 目录和 Mac Apple 芯片 `.app` 压缩包，结果在 Actions 的 Artifacts 下载。
- 推送 `v*` 标签：自动创建 GitHub Release，并上传 Windows / Mac 压缩包。

语法检查：

```bash
node --check uploader-server-v1.js
node --check step1-gallery-upload-api-v1.js
node --check step2-batch-design-api-v1.js
node --check step3-temu-export-api-v1.js
node --check ui/app-v24.js
```
